# Edge 翻译插件 MVP 产品说明

## 1. 项目目标

本项目的第一版目标是开发一个基于 Microsoft Edge 的浏览器翻译插件，用于在网页正文中提供沉浸式双语阅读体验。

本期 MVP 的核心效果如下：

- 用户点击页面内悬浮按钮后，开始翻译当前页面正文
- 原文内容保持不变
- 译文以段落形式插入在对应原文下方
- 用户无需离开当前页面即可阅读双语内容
- 用户可通过扩展选项页配置翻译服务

## 2. 目标用户

- 阅读英文博客、技术文档、新闻、教程的中文用户
- 希望保留原文上下文、同时快速理解内容的用户
- 接受手动触发翻译、优先追求结果可用性的早期用户
- 希望接入自有 OpenAI 兼容模型服务的开发者或重度用户

## 3. MVP 范围

### 3.1 本期要做

- 支持 Edge 浏览器扩展形态运行
- 支持页面内悬浮按钮触发整页正文翻译
- 支持悬浮按钮设置面板中的目标语言切换
- 支持快捷键触发翻译，默认 `Ctrl+Shift+A`
- 支持扩展选项页配置运行时 API 参数
- 支持在扩展选项页执行连接测试
- 优先识别网页正文区域，而不是扫描整个页面
- 支持提取正文中的常见段落节点
- 支持调用 OpenAI 兼容的大模型 API 执行翻译
- 支持将译文插入到原文下方
- 支持避免同一段重复插入译文
- 支持基础进度反馈、局部失败标记、失败重试
- 支持视口懒加载翻译和基础 SPA 新内容监听

### 3.2 本期不做

- 划词翻译
- 右键菜单翻译
- 图片 OCR 翻译
- PDF 翻译
- 多引擎切换
- 专业术语库
- 用户账号体系
- 云端同步
- 复杂缓存体系
- 远程托管用户配置

## 4. 用户流程

### 4.1 首次配置流程

1. 用户安装扩展
2. 用户打开扩展选项页
3. 填写 `API Base URL`、`API Key`、`Model`
4. 点击“连接测试”验证当前配置
5. 保存配置
6. 扩展后续翻译请求优先使用该运行时配置

### 4.2 主翻译流程

1. 用户打开一篇文章或文档页面
2. 页面内出现一个可拖拽的悬浮翻译按钮
3. 用户点击按钮，或者按下快捷键开始翻译
4. 插件识别正文主内容区域
5. 插件提取正文段落并分批发送到翻译服务
6. 翻译结果返回后，插件将译文插入对应原文下方
7. 用户在原页面中阅读原文和译文

### 4.3 页面内设置流程

1. 用户将鼠标悬停在悬浮按钮附近
2. 展开设置面板
3. 用户可切换目标语言、调整按钮显示方式、执行翻译或移除译文

### 4.4 异常流程

1. 如果页面没有识别到足够的正文内容，不进行注入
2. 如果某些批次翻译失败，仅标记对应段落失败，不中断整页其他段落
3. 如果用户再次点击翻译，默认跳过已翻译段落，避免重复插入
4. 如果运行时配置缺失，则提示用户去 options 页配置 API 参数
5. 如果连接测试失败，则在 options 页展示错误信息，用户可直接修改后重试

## 5. 交互要求

### 5.1 悬浮按钮最小交互

- 一个页面内悬浮按钮，点击即翻译
- 一个悬停后展开的设置入口
- 一个目标语言选择控件，默认简体中文
- 一个移除本页译文按钮
- 一个顶部进度条，用于显示翻译进度

### 5.2 选项页最小交互

- 一个 `API Base URL` 输入框
- 一个 `API Key` 输入框
- 一个 `Model` 输入框
- 一个保存配置按钮
- 一个连接测试按钮
- 一个清空配置按钮
- 一个简单状态提示区域

### 5.3 页面内展示

- 原文不做修改
- 每个译文块插入在对应原文块之后
- 译文样式与原文有清晰区分，但不过度破坏页面原有视觉层次
- 已翻译内容允许统一移除

## 6. 支持页面类型

优先支持以下页面：

- 博客文章页
- 技术文档页
- 教程页
- 资讯文章页

不保证第一版效果的页面：

- 社交媒体信息流
- 高动态单页应用
- 富交互后台系统
- 电商列表页
- 大量评论混排页面

## 7. 成功标准

- 可以在 Edge 中正常安装和运行
- 可以通过 options 页成功保存运行时 API 配置
- 可以通过 options 页成功完成连接测试
- 能在常见文章页面识别出主要正文区域
- 能将大多数正文段落成功翻译并插入原文下方
- 页面不出现明显结构破坏
- 翻译失败时不会导致整个页面功能中断

## 8. 后续演进方向

- 更完整的配置校验和错误提示
- 更稳健的结构化翻译协议
- 划词翻译
  - 支持按住 Ctrl 的划词翻译
  - 支持划词结果小弹窗
- 右键菜单翻译
- 图片 OCR 翻译
- 缓存与术语定制
- 多语言和多模型支持
