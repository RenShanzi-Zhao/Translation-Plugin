# Edge 翻译插件 MVP 产品说明

## 1. 项目目标

本项目的第一版目标是开发一个基于 Microsoft Edge 的浏览器翻译插件，用于在网页正文中提供沉浸式双语阅读体验。

本期 MVP 的核心效果如下：

- 用户点击插件按钮后，开始翻译当前页面正文
- 原文内容保持不变
- 译文以段落形式插入在对应原文下方
- 用户无需离开当前页面即可阅读双语内容

## 2. 目标用户

本期主要面向以下用户：

- 阅读英文博客、技术文档、新闻、教程的中文用户
- 希望保留原文上下文、同时快速理解内容的用户
- 接受手动触发翻译、优先追求结果可用性的早期用户

## 3. MVP 范围

### 3.1 本期要做

- 支持 Edge 浏览器扩展形态运行
- 支持用户点击扩展 Popup 中的按钮，翻译当前页面
- 优先识别网页正文区域，而不是扫描整个页面
- 支持提取正文中的常见段落节点
- 支持调用自有大模型 API 执行翻译
- 支持将译文插入到原文下方
- 支持避免同一段重复插入译文
- 支持最基础的失败提示和局部重试能力

### 3.2 本期不做

- 划词翻译
- 右键菜单翻译
- 图片 OCR 翻译
- PDF 翻译
- 自动滚动懒加载翻译
- SPA 动态内容自动监听
- 多引擎切换
- 专业术语库
- 用户账号体系
- 云端同步
- 复杂缓存体系

## 4. 用户流程

### 4.1 主流程

1. 用户打开一篇文章或文档页面
2. 用户点击扩展图标，打开 Popup
3. 用户点击“翻译当前页面”
4. 插件识别正文主内容区域
5. 插件提取正文段落并分批发送到翻译服务
6. 翻译结果返回后，插件将译文插入对应原文下方
7. 用户在原页面中阅读原文和译文

### 4.2 异常流程

1. 如果页面没有识别到足够的正文内容，提示“未检测到可翻译正文”
2. 如果某些批次翻译失败，仅标记对应段落失败，不中断整页其他段落
3. 如果用户再次点击翻译，默认跳过已翻译段落，避免重复插入

## 5. 交互要求

### 5.1 Popup 最小交互

Popup 第一版仅保留最小功能：

- 一个“翻译当前页面”按钮
- 一个“移除本页译文”按钮
- 一个目标语言选择控件，默认简体中文
- 一个简单状态区域，用于显示“翻译中”“完成”“失败”

### 5.2 页面内展示

- 原文不做修改
- 每个译文块插入在对应原文块之后
- 译文样式与原文有清晰区分，但不过度破坏页面原有视觉层次
- 已翻译内容允许统一移除

## 6. 支持页面类型

优先支持以下页面：

- 博客文章页
- 技术文档页
- 教程页
- 资讯文章页

不保证第一版效果的页面：

- 社交媒体信息流
- 高动态单页应用
- 富交互后台系统
- 电商列表页
- 大量评论混排页面

## 7. 成功标准

MVP 阶段的成功标准如下：

- 可以在 Edge 中正常安装和运行
- 能在常见文章页面识别出主要正文区域
- 能将大多数正文段落成功翻译并插入原文下方
- 页面不出现明显结构破坏
- 翻译失败时不会导致整个页面功能中断

## 8. 后续演进方向

本期 MVP 验证通过后，后续可逐步增加：

- 视口懒加载翻译
- 划词翻译
- 右键菜单翻译
- 图片 OCR 翻译
- SPA 动态节点监听
- 缓存与术语定制
- 多语言和多模型支持
